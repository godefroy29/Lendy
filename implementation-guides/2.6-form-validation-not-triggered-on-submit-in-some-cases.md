# 2.6 Form Validation Not Triggered on Submit in Some Cases

**Priority:** Low  
**Category:** Bug Fixes  
**Status:** ✅ Implemented  
**Difficulty:** 2 ⭐⭐  
**Modification Location:** Flutter Code

## Description

In `CreateItemScreen`, form validation might not show all errors if user submits without interacting with fields.

## Implementation Details

- Ensure `_formKey.currentState!.validate()` is called before submission
- Use `autovalidateMode: AutovalidateMode.onUserInteraction` or `AutovalidateMode.always` in form fields
- Show all validation errors at once
- Scroll to first error field if form is long

## Files to Modify

- **Modify:** `lendy/lib/presentation/screens/create_item_screen.dart` (form validation)

## Dependencies

None

## Implementation Steps

1. Add `autovalidateMode: AutovalidateMode.onUserInteraction` to Form widget
2. Ensure `validate()` is called before submission
3. Add `autovalidateMode` to individual `TextFormField` widgets
4. Test form submission without interacting with fields
5. Ensure all errors are displayed
